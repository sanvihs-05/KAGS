## Validation Framework

The validation framework checks the following aspects:

### 1. **Schema Validation** 
- Validates presence of essential sections (functions, behaviors, structures, layout, scores)
- Checks data types and required fields
- Ensures JSON structure integrity

### 2. **Functional Validation**
- Verifies all functions have valid priority scores (0-1 range)
- Checks spatial requirement constraints (min ≤ preferred ≤ max area)
- Calculates total area requirements
- Validates priority distribution across functions

### 3. **Behavioral Validation**
- Checks behavioral performance targets against actual values
- Validates tolerance levels are respected
- Calculates satisfaction scores for thermal, acoustic, lighting, and ventilation behaviors
- Ensures metric values are within realistic ranges

### 4. **Structural Validation**
- Validates U-values (thermal transmittance) against typical ranges:
  - External walls: 0.15-0.35 W/m²K
  - Windows: 0.70-2.5 W/m²K
  - Roof: 0.10-0.25 W/m²K
  - Floor slabs: 0.15-0.40 W/m²K
- Checks acoustic ratings (STC values)
- Validates material properties

### 5. **Layout Validation**
- Verifies layout metrics are in valid range (0-1):
  - Compactness
  - Circulation efficiency
  - Adjacency satisfaction
- Checks room area calculations (width × height = area)
- Validates total area vs. sum of room areas
- Calculates area utilization percentage

### 6. **Score Validation**
- Ensures all individual scores are in range [0, 1]
- Validates composite score calculation (average of individual scores)
- Checks consistency between top-level and nested scores
- Identifies outliers and anomalies

### 7. **Cross Validation**
- Matches function requirements with actual layout room areas
- Validates alignment between different sections
- Checks for consistency across the entire design

## Appendix: Validation Framework Usage

### Custom Validation Checks

You can extend the framework by adding custom validation methods:

```python
class CustomValidator(LayoutValidator):
    def _validate_building_codes(self, data):
        """Add your building code validation logic"""
        issues = []
        
        # Example: Check minimum bedroom size (9 m² per code)
        for room in data.get("layout", {}).get("rooms", {}).values():
            if "bedroom" in room.get("name", "").lower():
                if room.get("area", 0) < 9:
                    issues.append(f"{room['name']} below minimum size (9 m²)")
        
        return {
            "status": "PASS" if not issues else "FAIL",
            "issues": issues
        }
```

### Batch Processing

For validating large numbers of designs:

```python
import glob
from validation_framework import LayoutValidator

validator = LayoutValidator()
all_results = []

for file in glob.glob("**/*.json", recursive=True):
    result = validator.validate_file(file)
    all_results.append(result)
    
    # Only print failures
    if "[X]" in result["overall_status"]:
        validator.print_summary(result)
```
